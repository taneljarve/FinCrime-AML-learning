# Day 1: Project Foundation

This is an example of what a properly formatted day looks like using Daily Structured Learning methodology.

## Learning Objectives

By the end of today, students will:
- ✅ Understand project structure and how to navigate
- ✅ Complete setup and installation requirements
- ✅ Make their first commit to git

## Key Concepts

### Concept A: Project Organization
The project is organized to support self-paced learning with git tracking. Each day has:
- A markdown file with learning objectives
- TODOs that build on each other
- Resources for independent learning
- Questions to test understanding

### Concept B: The TODO System
Each TODO has a specific format:
- **What to do**: Clear, actionable instruction
- **Why**: Explains the significance
- **Validation**: How to know you're done
- **Hint**: Nudges you toward the solution without spoiling it

## Today's Tasks

### TODO 1: Complete Initial Setup
**What to do:** 
- Clone or create the project folder
- Install all required dependencies
- Verify installation with a simple test command

**Why:** 
Without proper setup, later tasks will fail. This validates your environment is correct.

**Validation:** 
Run the verification command - if you see the expected output, you're done.

**Hint:** 
Check the README.md in the project root for step-by-step setup instructions.

### TODO 2: Understand Project Structure
**What to do:** 
- Navigate through all folders in the project
- Read file names and comments
- Create a quick mental map or written list of what each folder contains

**Why:** 
Understanding structure helps you know *where* to add code and *why* it goes there.

**Validation:** 
You can explain: "Config files go in X, code in Y, tests in Z"

**Hint:** 
Look at the main README.md - most projects have a folder structure diagram.

### TODO 3: Make Your First Commit
**What to do:** 
- Make a small change (e.g., add your name to a file)
- Run `git add .`
- Run `git commit -m "Complete Day 1 setup"`
- Run `git log` to verify your commit appears

**Why:** 
Git commits track your learning progress. Each day's work should be saved.

**Validation:** 
Your commit appears in `git log` with today's date and message.

**Hint:** 
If you're unsure about git commands, search "git basics tutorial" - every good tutorial explains add/commit/log.

### Answer 1: Project Structure Purpose
The project structure organizes code, configuration, and learning materials so students can find and understand each component. It separates concerns (e.g., config separate from code) making the project easier to navigate and scale.

### Answer 2: Verify Setup
You verify setup by:
- Running the installation verification command
- Checking that required dependencies are installed
- Running a simple "hello world" test
- Looking for expected output (no errors)

If all these pass, your environment is correctly set up.

### Answer 3: Why Git Commits Track Learning
Git commits create a timestamped record of your work. This lets you:
- See your progress over 14 days (one commit per day)
- Revert if you break something
- Share your work with instructors
- Understand how each day built on the previous
